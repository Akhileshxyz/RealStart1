# RealStart Backend: Frontend (Public) User Authentication APIs

## Overview
This document outlines the authentication APIs used by the frontend public portal (Buyer role). These APIs are managed in `app/api/v1/public_auth.py`.

---

## 1. User Registration
- **Endpoint:** `POST /api/v1/auth/register`
- **Description:** Creates a new account. All registrations from this endpoint are automatically assigned the `BUYER` role.
- **Request Fields (JSON):**
    - `full_name`: string (e.g., "John Doe")
    - `email`: string (valid email format, must be unique)
    - `password`: string (must meet complexity requirements)

### Password Validation Rules:
- Minimum **8 characters**.
- At least one **uppercase letter**.
- At least one **lowercase letter**.
- At least one **number**.
- At least one **special character** (`!@#$%^&*(),.?":{}|<>`).

---

## 2. User Login (OAuth2 Compatible)
- **Endpoint:** `POST /api/v1/auth/login`
- **Method:** `application/x-www-form-urlencoded` (Standard OAuth2)
- **Request Fields (Form Data):**
    - `username`: (Buyer's email)
    - `password`: (Buyer's password)

### Flow & Restrictions:
- **Rate Limiting:** Restricted to **5 attempts per minute** via `slowapi`.
- **Role Constraint:** Only allows users with the `BUYER` role. Admins/Developers are rejected with a 403.
- **Response Shape:**
    ```json
    {
      "access_token": "JWT_TOKEN_HERE",
      "token_type": "bearer",
      "user": {
        "id": "UUID",
        "email": "user@example.com",
        "full_name": "John Doe",
        "role": "BUYER",
        "is_active": true
      }
    }
    ```

---

## 3. Current User Profile
- **Endpoint:** `GET /api/v1/auth/me`
- **Authorization:** Requires `Authorization: Bearer <JWT_TOKEN>`
- **Description:** Returns the complete user profile for the current authenticated user.

---

## 4. User Logout
- **Endpoint:** `POST /api/v1/auth/logout`
- **Authorization:** Requires `Authorization: Bearer <JWT_TOKEN>`
- **Description:** Invalidate the current session by blacklisting the token in Redis for 7 days.

---

## 5. Get User Profile (Extended)
- **Endpoint:** `GET /api/v1/users/me`
- **Authorization:** Requires `Authorization: Bearer <JWT_TOKEN>`
- **Description:** Returns full user profile including all extended fields.
- **Response Fields:**
    - `id`: UUID
    - `email`: string
    - `full_name`: string
    - `phone`: string (nullable)
    - `photo_url`: string (nullable)
    - `region`: string (nullable)
    - `role`: string (e.g., "BUYER")

---

## 6. Update User Profile
- **Endpoint:** `PATCH /api/v1/users/me`
- **Authorization:** Requires `Authorization: Bearer <JWT_TOKEN>`
- **Description:** Updates one or more fields on the current user's profile. All fields are optional. Accepts `multipart/form-data` to support photo uploads.
- **Request Fields (Form Data — all optional):**
    - `full_name`: string
    - `email`: string (must be unique; validates against existing accounts)
    - `phone`: string (mobile number)
    - `photo`: File (Actual file upload. Supported: .jpg, .jpeg, .png, .heic, .heif)
    - `region`: string (e.g., "Bengaluru", "Mumbai")
- **Notes:**
    - Email uniqueness is validated — returns `400` if already taken.
    - Only fields provided in the request are updated.
    - Uploading a `photo` will process the file, store it locally, and automatically update the user's `photo_url` returned in future `/me` queries.

---

## 7. Get User Notifications
- **Endpoint:** `GET /api/v1/users/me/notifications`
- **Authorization:** Requires `Authorization: Bearer <JWT_TOKEN>`
- **Description:** Returns paginated list of user notifications, ordered by newest first.
- **Query Parameters:**
    - `page` (int, default=1): Page number.
    - `size` (int, default=20): Items per page (max 100).
    - `unread_only` (bool, default=false): Filter only unread notifications.
- **Response Shape:**
    ```json
    {
      "items": [
        {
          "id": "UUID",
          "title": "New Message",
          "body": "Your property has received a new lead.",
          "type": "SYSTEM",
          "is_read": false,
          "action_url": null,
          "created_at": "2023-11-20T10:00:00Z"
        }
      ],
      "total": 5,
      "page": 1,
      "size": 20,
      "pages": 1
    }
    ```

---

## 8. Update Notification (Mark Read/Unread)
- **Endpoint:** `PATCH /api/v1/users/me/notifications/{notification_id}`
- **Authorization:** Requires `Authorization: Bearer <JWT_TOKEN>`
- **Description:** Mark a specific notification as read or unread.
- **Request Fields (JSON):**
    - `is_read`: boolean (default=true)

---

## 9. Mark All Notifications as Read
- **Endpoint:** `POST /api/v1/users/me/notifications/mark-all-read`
- **Authorization:** Requires `Authorization: Bearer <JWT_TOKEN>`
- **Description:** Marks all unread notifications for the user as read.
- **Response:** `{"message": "All notifications marked as read"}`

---

## 10. Toggle Wishlist Item
- **Endpoint:** `POST /api/v1/wishlist/toggle`
- **Authorization:** Required
- **Description:** Adds or removes a property/project from the user's wishlist in a single toggle action.
- **Request Fields (JSON):**
    - `property_id`: UUID (ID of the project)
    - `type`: string (optional, defaults to "project")
- **Response:**
    ```json
    {
      "status": "success",
      "message": "Property added to wishlist",
      "action": "added",
      "count": 12
    }
    ```

---

## 11. List Wishlist Items
- **Endpoint:** `GET /api/v1/wishlist`
- **Authorization:** Required
- **Description:** Retrieves all wishlisted projects for the user.
- **Query Parameters:**
    - `page` (int, default=1)
    - `size` (int, default=10)
- **Response:**
    ```json
    {
      "status": "success",
      "data": [
        {
          "id": "uuid",
          "name": "North Gate Enclave",
          "location": "Thanisandra",
          "distance": "1.4 km",
          "price_sqft": 8100,
          "image_url": "/uploads/...",
          "added_at": "2024-03-27T10:00:00Z"
        }
      ]
    }
    ```

---

## 12. Clear Wishlist
- **Endpoint:** `DELETE /api/v1/wishlist`
- **Authorization:** Required
- **Description:** Removes all items from the user's wishlist.
- **Response:** 204 No Content
